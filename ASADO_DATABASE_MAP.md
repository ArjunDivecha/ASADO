# ASADO Database Map

**File:** `Data/asado.duckdb` (DuckDB v1.5, 2.7 GB) · **35 objects** (32 base tables + 3 views) · **~79.0 M base-table rows** · **31–43 countries** depending on table · **history 1950 → 2100** (most realized data 2000 → 2026‑05).

**ASADO** is a hybrid data‑collection and research platform for a fixed **34‑country macro/equity "T2 universe."** It pulls macro, governance, risk, climate, trade, commodity, news‑sentiment, prediction‑market and Bloomberg‑terminal data from ~26 free sources + the World Bank "Pink Sheet" + 28 Bloomberg variables, aligns everything to the 34 T2 country names, and stores it in a hybrid **DuckDB (analytical warehouse) + Neo4j (knowledge graph)** stack. The purpose is **country rotation / factor timing / regime analysis**: explanatory features feed optimizer pipelines that produce top‑20%‑of‑countries factor‑portfolio returns. The governing doctrine is **"returns are the source of truth"** — commodity, GDELT, Bloomberg, macro, prediction‑market and graph layers are *context* unless explicitly joined back to a return surface.

> ⚠️ **Generated by automated deep‑dive (2026‑05‑29).** Every count, date range and example below was verified against the live `asado.duckdb` at that time. Where the project docs (README/CLAUDE.md/PRDs) disagree with the live DB, this map trusts the **live DB** and flags the mismatch. See [§5 Gotchas](#5-gotchas--caveats).

## 1. Quick orientation

### The tidy/long panel pattern
Nearly every table shares one schema:

| Column | Type | Example |
|---|---|---|
| `date` | DATE | `2020-01-01` (first‑of‑month for monthly data) |
| `country` | VARCHAR | `Brazil` (exact T2 name) |
| `value` | DOUBLE | `189.37` |
| `variable` | VARCHAR | `Best PE _CS`, `BIS_Credit_GDP_Gap`, `WB_CMDTY_CRUDE_BRENT_LEVEL` |
| `source` | VARCHAR | `t2`, `gdelt`, `bloomberg`, `wb_commodity`, … |

This is an EAV/panel layout: instead of 2,000 columns there are **2,000+ distinct `variable` values** in long form. To "see what a table holds," `SELECT DISTINCT variable` — not the column list.

### `_CS` vs `_TS` — the central concept
Every normalized signal exists in two flavors:
- **`_CS` (Cross‑Sectional):** on each **date**, z‑score the variable **across the 34 countries**. "How attractive is this country vs its peers right now." Scale: mean ≈ 0, std ≈ 0.98–1.0.
- **`_TS` (Time‑Series):** for each **country**, z‑score against an **expanding window of its own history** (min 12 obs). "How extreme is this reading vs this country's own norms." Scale slightly off‑center (expanding stats drift).
- **Sign convention:** ~19 "lower‑is‑better" variables (valuation ratios, `REER`, `RSI14`, `Debt to GDP`, short‑term reversal, `Bloom Country Risk`, …) are multiplied by −1 *after* normalization, so **higher normalized value = more attractive, universally.** A `0` often means "no signal" (zero‑std or post‑fill imputation), **not** "neutral mid‑rank."

### The 34 T2 names (exact strings required everywhere)
```
Australia, Brazil, Canada, Chile, ChinaA, ChinaH, Denmark, France, Germany,
Hong Kong, India, Indonesia, Italy, Japan, Korea, Malaysia, Mexico, NASDAQ,
Netherlands, Philippines, Poland, Saudi Arabia, Singapore, South Africa,
Spain, Sweden, Switzerland, Taiwan, Thailand, Turkey, U.K., U.S.,
US SmallCap, Vietnam
```
Mapping rules: "China" → **both ChinaA and ChinaH**; "United States" → **U.S., NASDAQ, AND US SmallCap**. `country_reference` collapses these to **31 sovereigns** (sleeves folded in); `daily_calendar` and the factor tables carry all **34**.

### The signal → return → attribution flow
```
raw levels (t2_raw)  ──normalize (CS+TS)──▶  signals (t2_master / t2_factors_daily)
                                                   │  optimizer ranks countries, top‑20% bucket
                                                   ▼
   factor_returns (monthly P&L) ◀──┬──▶ factor_top20_membership (which countries are long)
   factor_returns_daily (daily P&L)│
                                   ▼  JOIN on (date, factor, source)
                          country_factor_attribution (VIEW): contribution = weight × factor_return
```

### Where to find things
- **"What does the warehouse know about variable X?"** → `variable_meta` (654 rows, but only covers daily + commodity + predmkt tables; the full ~3,000‑var catalog is in `Data/cache/query_assistant/variable_catalog.json` and `docs/factor_reference.md`).
- **Country code crosswalk** → `country_reference` (ISO2/ISO3/IMF/OECD/BIS/WB/EPU/GPR).
- **Master raw cross‑source surface** → `unified_panel` (VIEW, 17.4 M rows, 10 tables unioned).
- **The live tradeable strategy** → the **8 optimizer‑selected factors** (`is_optimizer_selected=TRUE` in `variable_meta`): `120DTR_TS`, `120MA Signal_CS`, `120MA Signal_TS`, `20DTR_CS`, `20DTR_TS`, `REER_CS`, `RSI14_CS`, `RSI14_TS` — all in `t2_factors_daily`.

### A. T2 Factor Core — the spine

The T2 system is a multi‑factor country‑equity allocation model. Flow: **raw Bloomberg levels → normalized signals → factor‑portfolio returns → country attribution.** Universe: exactly **34 fixed countries**, 2000‑present, **no entry/exit** (survivorship‑bias caveat; early EM history forward‑filled).

| Object | Type | Rows | Vars | Date range | Grain (1 row =) |
|---|---|---:|---:|---|---|
| `t2_raw` | TABLE | 485,582 | 53 | 2000‑02 → 2026‑05 | date×country×variable, **raw monthly level** |
| `t2_master` | TABLE | 1,192,584 | 111 | 2000‑02 → 2026‑05 | date×country×variable, **normalized + forward returns** |
| `t2_factors_daily` | TABLE | 32,340,392 | 109 | 2000‑01 → 2026‑05‑07 | date×country×variable, **daily signal** |
| `t2_levels_daily` | TABLE | 13,698,294 | 47 | 2000‑01 → 2026‑04‑21 | date×country×variable, **raw daily level** |
| `factor_returns` | TABLE | 277,116 | 1,678 (3 sources) | 2000‑02 → 2026‑04 | date×factor×source, **monthly portfolio return** |
| `factor_returns_daily` | TABLE | 1,293,492 | 178 (2 sources) | 2000‑01 → 2026‑05‑07 | date×factor×source, **daily portfolio return** |
| `factor_top20_membership` | TABLE | 2,104,980 | 1,681 | 2000‑02 → 2026‑05 | date×country×factor×source, **weight>0 only** |
| `country_factor_attribution` | VIEW | 2,087,921 | — | 2000‑02 → 2026‑04 | date×country×factor×source, **contribution** |
| `t2_factors_monthly_from_daily` | VIEW | 1,039,973 | 109 | 2000‑01 → 2026‑05 | month×country×variable EOM snapshot |

- **`t2_raw`** — raw Bloomberg/workbook monthly levels (53 vars), e.g. `Best PE` (0.05–137), `Inflation` (−5.3 to 37), `REER` (64–186), `RSI14` (18–95), `1MTR` (trailing 1‑mo return, −0.41 to +0.49). Raw inputs, *not* normalized.
- **`t2_master`** — the processed canonical panel: 111 vars = **5 forward returns** (`1MRet, 3MRet, 6MRet, 9MRet, 12MRet`) + **53 `_CS`** + **53 `_TS`**. Raw levels are dropped (only `Best PE _CS/_TS`, no bare `Best PE`). **⚠️ Forward‑return trap:** `1MRet` at month *t* = raw `1MTR` at month *t+1* (verified). It is the *forward* dependent variable, not a same‑period return.
- **`t2_factors_daily`** — daily normalized signals (the event‑study workhorse): 52 `_CS` + 52 `_TS` + 5 daily returns (`1DRet, 5DRet, 20DRet, 60DRet, 120DRet`). Daily naming differs from monthly (`20DTR` vs `1MTR`, `120DTR` vs `12MTR`, `120‑5DTR` vs `12‑1MTR`); use `variable_meta.monthly_equivalent` to map.
- **`t2_levels_daily`** — daily **raw** levels (47 vars) when you need actual prices/ratios not z‑scores: `PX_LAST`, `RSI14`, `REER`, `10Yr Bond` (yields %), `Inflation`, etc. ~2 weeks staler than the signals table (max 2026‑04‑21).
- **`factor_returns`** (monthly realized P&L, schema `date,factor,value,source`) — long‑only top‑20%‑bucket portfolio return per factor. **3 sources:** `t2_optimizer` (82 factors, T2 style), `econ_optimizer` (274 factors, BIS/macro names like `BIS_Credit_GDP_Gap_CS`), `gdelt_optimizer` (1,322 factors, news names). Value ≈ percent‑scale (std ~2, range ±15).
- **`factor_returns_daily`** — daily P&L, **2 sources only:** `t2_optimizer_daily` (104 factors) + `gdelt_optimizer_daily` (74). **No econ daily.** Value ≈ daily‑scale (std ~1 / ~0.36).
- **`factor_top20_membership`** — sparse (weight>0 only): which countries are **long** each factor and their weight (sum ≈ 1.0 per date×factor; soft 15–25% rank taper from "Step Three"). ~7 countries long per T2 factor. **Encodes only the long side — not shorts.**
- **`country_factor_attribution`** (VIEW) — `contribution = weight × factor_return`, joining membership to **monthly** `factor_returns` (verified: 0 of 2.09 M rows deviate). Top‑20% bucket only, **not** a full portfolio decomposition and **not** causal. No daily attribution surface exists.

**The 8 live optimizer‑selected factors** (`is_optimizer_selected=TRUE`, all in `t2_factors_daily`, read from `T2_Optimizer_Top.xlsx`): `120DTR_TS`, `120MA Signal_CS`, `120MA Signal_TS`, `20DTR_CS`, `20DTR_TS`, `REER_CS`, `RSI14_CS`, `RSI14_TS`. All 8 have full daily P&L in `factor_returns_daily`; only 5 appear in monthly `factor_returns` (the 3 DTR‑named ones are daily‑native naming).

### B. GDELT News / Sentiment

Country‑day news sentiment & attention from GDELT's Global Knowledge Graph (GKG v2), built by an **external** pipeline and ingested via `setup_duckdb.py` (no ASADO collector runs GDELT). Three tables:

| Table | Rows | Freq | Vars | Countries | Date range |
|---|---:|---|---:|---|---|
| `gdelt_raw_daily` | 943,652 | daily | 45 **wide cols** | **249** (raw GDELT entities) | 2015‑02‑18 → 2026‑05‑07 |
| `gdelt_factors_daily` | 10,085,794 | daily | 75 (tidy) | 34 (T2) | 2015‑06‑24 → 2026‑05‑08 |
| `gdelt_panel` | 5,622,818 | **MONTHLY** | **1,323** | 34 (T2) | 2015‑09‑01 → 2026‑03‑01 |

- **`gdelt_raw_daily`** (WIDE, not tidy) — one row per country‑day, 45 columns: volume/attention (`n_articles`, `local/foreign_n_articles`, `local_attention_share`), tone (`tone_mean` avg −0.78, `tone_p10/50/90`, `tone_dispersion`), local‑vs‑foreign tone split, themes (`top_themes_json`, often empty), **GKG fetch‑completeness QA columns** (`gkg_fetch_share` avg 0.998, `day_status` ok/partial/NULL), and pre‑computed signals (`country_news_sentiment`, `country_news_risk`, `attention_shock`, z‑scores). This is the **249‑country off‑universe bridge** (incl. Iran, North Korea, etc.); keyed by FIPS `country_code_gdelt` + `country_iso3`, China = single `CHN` row (no sleeve fanout yet).
- **`gdelt_factors_daily`** — 75 tidy daily signals = 37 base names × `_CS`/`_TS` + `1DRet`. Families: sentiment (fast/slow/trend + z), attention (fast/slow/trend, shock), risk/dispersion, local‑foreign tone split. **Headline composites:** `monthly_metronome` = 0.35·sent_fast_z + 0.20·sent_slow_z + 0.20·sent_trend_z + 0.15·attn_fast_z − 0.10·risk_fast_z (higher = more positive outlook); `monthly_risk` = the risk‑off mirror; `monthly_defensive` = −`monthly_risk`. EWMA spans: fast=5, slow=20, risk=10.
- **`gdelt_panel`** — **MONTHLY** (first‑of‑month), the *only* table with the deep feature families: `theme_*` (1,032 GKG theme‑share vars, e.g. `theme_PROTEST_share`, `theme_ARMEDCONFLICT_share`) and `gcam_*` (150 GDELT emotional/psychometric dictionary vars: VADER, Loughran‑McDonald, Moral Foundations, ANEW, etc.). Plus the `core_signal` family (140) and `1MRet`. Structure: 661 `_CS` + 661 `_TS` + `1MRet`.

**`1DRet`/`1MRet` are bit‑exact aliases of the T2 country returns** (copied in as the optimizer's dependent variable) — **not** an independent return source. The actual GDELT‑optimizer P&L lives in `factor_returns_daily` (source `gdelt_optimizer_daily`).

### C. World Bank Commodities

Built from the official World Bank "Pink Sheet" workbook (`CMO-Historical-Data-Monthly.xlsx`, source dated 2026‑05‑04). Chain: **raw prices/indices → meta (catalog + projection flag) → features (7 trailing transforms) → factor_panel (broadcast to 34 countries).** All span **1960‑01 → 2026‑04** (monthly). **Explanatory‑only by design** — never fed into optimizer‑output tables.

| Table | Rows | Grain | Notes |
|---|---:|---|---|
| `wb_commodity_prices` | 50,099 | date×commodity_code | 71 benchmark **nominal‑USD** prices, 10 categories, 0 NULLs |
| `wb_commodity_indices` | 12,736 | date×index_code | 16 aggregate indices (**2010=100**), e.g. iENERGY, iAGRICULTURE |
| `wb_commodity_meta` | 87 | series_code | catalog; `is_projected_to_factor_panel` = the broadcast switch |
| `wb_commodity_features` | 435,618 | date×series_code×feature | 7 trailing transforms, commodity axis preserved |
| `wb_commodity_factor_panel` | 9,600,274 | date×country×variable | 371 vars broadcast to 34 countries |

- **7 features:** `level`, `mom_pct`, `yoy_pct`, `ret_3m_pct`, `ret_12m_pct`, `vol_12m`, `z_36m` (z winsorized at ±5.916). Point‑in‑time safe (trailing only).
- **Broadcast math:** 53 projected series (all 16 indices + 37 high‑signal prices) × 7 features = **371 variables**, each **identical across all 34 countries** for a given date (`is_global_broadcast=TRUE` for 100% of rows). These are global prints, not country‑specific numbers.
- **Naming:** `WB_CMDTY_{CODE}_{SUFFIX}` with irregular suffixes (`mom_pct→_MOM`, `yoy_pct→_YOY`, `ret_12m_pct→_RET_12M`, …). E.g. `WB_CMDTY_CRUDE_BRENT_LEVEL`, `WB_CMDTY_COPPER_YOY`.
- **Footprint:** these 371 vars are **55.1% of all unified_panel rows** and the largest block of `variable_meta` (371/654 = 56.7%).

| `external_factors` | 112,677 | 35 | 34 | 1985‑01 → 2026‑03 | gpr, worldbank, bis_reer, epu, oecd, bis_credit, bis_property |

- **`bloomberg_factors`** — 17 `BBG_*` sovereign/macro (govt bond yields 2Y–30Y, OIS, CDS_5Y, MIPD default prob, breakevens, WIRP, ECFC consensus, PMI, M2_YoY, Debt/GDP) + 11 `MS_*` ETF/passive‑flow market‑structure vars. **Very uneven coverage:** `BBG_Debt_GDP_Ratio` = 1 country (Indonesia), `BBG_Breakeven_10Y` = 6, `BBG_CDS_5Y` = 15 (mostly EM). Source tag always `bloomberg`.
- **`imf_factors`** — trade (`imf_itg`, largest), CPI, rates, WEO forecasts, FX, BOP. **1,218 forward‑dated rows (all `imf_weo`) out to 2031‑12.**
- **`external_factors`** — Program 1 core: GPR (+ 3 broadcast `Global_GPR*`), EPU, BIS REER/credit‑gap/property, OECD CLI, 25 `WB_*` (6 governance indicators −2.5..+2.5 + macro/structural).
- **`extended_factors`** — Program 2 (widest panel): FRED markets (UST, VIX, USD index, HY OAS — US/global broadcasts), 26 ECB FX crosses, BIS policy rate/DSR, OECD BCI/CCI, UNDP HDI, ND‑GAIN climate, FAO, ILO, EIA, OFAC.
- **`macrostructure_factors`** — Program 5: bank soundness (`imf_fsi`), public‑debt structure (`qpsd`), central‑bank footprint, institutional/pension assets, US‑holder share (derived from the bilateral matrix), and transparent percentile composites (`MS_Investor_Base_Fragility`, `MS_Policy_Backstop`, etc., 0–100).
- **`demographics_dip`** — Demographic Inflation Pressure (UN WPP 2024 + Juselius‑Takats BIS WP 722). 4 vars (`DIP_abs`, `DIP_rel` — the tradeable cross‑sectional signal — `DIP_rel_chg_5y/10y`). **10,200 forward‑dated rows to 2100.** ⚠️ Uses the **paper's universe, NOT the T2 universe** (adds Austria/Belgium/Finland/Greece/Ireland/NZ/Norway/Portugal/Russia; drops ChinaH/HK/NASDAQ/Philippines/Saudi/Singapore/Taiwan/US SmallCap/Vietnam).
- **`bilateral_portfolio_matrix`** — **different grain:** (date, reporter_iso3, counterpart_iso3, instrument_type, amount_usd, shares…). `imf_pip` (annual benchmark, 50,570 rows) + `us_tic` (monthly US detail, 6,216 rows). **NOT in `unified_panel`.** ISO3 codes → join via `country_reference`. `is_official_sector` always FALSE (placeholder).

There is **no variable‑name overlap** among the six panels — they are partitioned by source family, not deduplicated. (`MS_` prefix is split: 11 ETF vars in `bloomberg_factors`, 26 fragility vars in `macrostructure_factors`, disjoint.)

### E. Prediction Markets — ⚠️ thin pilot, not production

Six tables ingesting curated Kalshi/Polymarket probabilities and bridging them to the T2 universe. **The entire layer is ONE snapshot (`2026‑05‑19`).** PRD V1 targets (≥150 markets, 14 unattended days) are **not** met.

| Table | Rows | Grain | Status |
|---|---:|---|---|
| `predmkt_market_meta` | 15 | market | 12 Polymarket + 3 Kalshi, all binary |
| `predmkt_outcome_meta` | 30 | outcome | all Yes/No; thresholds 100% NULL (no scalar markets) |
| `predmkt_daily` | 30 | date×market×outcome | **single snapshot 2026‑05‑19**, no time series |
| `predmkt_country_spillover` | 49 | market×country edge | 18 markets → 18 countries, signed elasticity −0.85..+0.85 |
| `predmkt_signals_daily` | 40 | date×signal×country | 14 derived signals (4 empty) |
| `predmkt_resolutions` | **0** | — | **EMPTY** — calibration never ran |

- **Categories:** tariff, cpi, fed_policy, oil_shock, regional_conflict (ME/Pacific), country_election, gdp_recession.
- **Spillover channels (8 used):** trade_partner, regional_proximity, usd_beta, oil_import_dependency, regional_proxy, em_beta, tech_supply_chain, oil_export. Convention: positive elasticity = "YES outcome lifts this country's equity."
- **Signals:** global (`fed_cut_count_expectation`, `oil_shock_prob_30d`, `hormuz_disruption_prob_90d`, `recession_prob_12m`, conflict premiums) + country (`predmkt_country_risk_composite` and `..._opportunity_composite` are exact negatives; positive risk = implied downside) + `tariff_intensity_by_country`. 4 signals are NULL/empty (cpi/unemployment nowcasts, eastern‑europe conflict).
- **`confidence_score` is a hardcoded 0.6** (resolutions empty → no calibration). Liquidity is real ($5.7K–$2.4M; some markets show $17.9M total volume), so the *seed* markets are genuine — there's just only one day of them.

### F. Reference / Meta / Events / Calendar / Unified Surface

The connective tissue.

- **`country_reference`** (31 rows) — ISO/source‑code crosswalk. **31 sovereigns, not 34:** ChinaH folds into ChinaA, NASDAQ + US SmallCap fold into U.S. (only those 2 parent rows have `has_market_sleeves=TRUE`; sleeves live in the `market_sleeve_variants`/`all_country_variants` columns). ChinaA is the one `graph_role='sovereign_proxy'`. Columns: `iso2, iso3, imf, oecd, bis, wb, fred_epu, gpr, epu_col, …`. Static dimension (no date). Join on `country` (T2 name) or `iso3`.
- **`variable_meta`** (654 rows) — the data dictionary for the **daily + commodity + predmkt** surfaces only (wb_commodity 371, t2_factors_daily 109, gdelt_factors_daily 75, t2_levels_daily 47, gdelt_raw_daily 38, predmkt_signals_daily 14). Columns: `variable, source_table, source_file, native_frequency (only 'monthly'|'D'), monthly_equivalent, is_normalized, category, is_optimizer_selected, freshness_expectation`. **8 rows `is_optimizer_selected=TRUE`** (the live factors). `monthly_equivalent` is meaningful in only 8 rows (daily→monthly return map); for commodities it just echoes the variable name. **Does NOT cover** the 7 monthly macro tables or `gdelt_panel`.
- **`daily_calendar`** (327,216 rows) — per‑country trading‑day flags, 34 countries, 2000‑01 → 2026‑05. Stores **every** calendar day (`is_trading_day` TRUE 222,076 / FALSE 105,140). Anchors event windows & month‑end snapshots.
- **`event_log`** (146 rows) — hand‑curated dated macro/geopolitical events, 1997‑07 → 2026‑05. PK `event_id`. Categories: central_bank 34, geopolitical 21, macro_release 18, regulatory 16, election 15, financial_crisis 15, oil_supply 14, trade_policy 13. Severity: high 92 / medium 54 / **no 'low'**. `is_global=TRUE` (98) → affects all 34 (`countries_affected` NULL); `is_global=FALSE` (48) → CSV of exact T2 names. Off‑universe actors (Iran, Russia) live in `tags`, never `countries_affected`. **No `end_date` is ever populated** (all point dates). Seeded from `config/event_log_seed.yaml`.
- **`unified_panel`** (VIEW, **17,412,105 rows, 2,026 vars, 43 countries**) — `UNION ALL` of **exactly 10 physical tables**: t2_master (`t2`), t2_raw (`t2_raw`), external_factors, extended_factors, gdelt_panel (`gdelt`), imf_factors, macrostructure_factors, wb_commodity_factor_panel, bloomberg_factors (cast DOUBLE, NULL‑filtered), demographics_dip. Exposes ~39 `source` sub‑labels. **Date range 1950 → 2100** (demographics/WEO projections — not realized data). **43 countries** because demographics_dip injects 9 non‑T2 names. **Deliberately excludes** optimizer outputs (`factor_returns*`, `factor_top20_membership`, `country_factor_attribution`), the daily tables, prediction markets, and the bilateral matrix — the **returns‑first cycle guardrail.**

## 4. Glossary of the critical conventions

- **`_CS`** = cross‑sectional z‑score (across countries, per date). **`_TS`** = time‑series z‑score (within country, expanding history).
- **Forward return** (`1MRet`…`12MRet` in t2_master): `1MRet@t = 1MTR@t+1`. The dependent variable, not trailing.
- **top‑20% bucket**: each factor's long portfolio = the most‑attractive ~15–25% of countries (soft taper), weights sum to 1.
- **broadcast / global variable**: same value for all countries on a date (all WB commodities, FRED markets, `Global_GPR*`). Never treat as country‑specific.
- **returns‑first guardrail**: optimizer *outputs* (factor_returns*) are never unioned into the input surfaces — prevents an input/output modeling cycle.

## 5. Gotchas & caveats (consolidated)

### Doc‑vs‑reality mismatches (the docs are partly stale)
1. **`feature_panel` and `normalized_panel` DO NOT EXIST in the live DB** — yet CLAUDE.md/README call `feature_panel` (claimed 31.6 M rows / 3,048 vars) "the most commonly queried" surface. The only `%panel%` objects are `unified_panel` (VIEW), `gdelt_panel`, `wb_commodity_factor_panel`. **Any query/code assuming `feature_panel` exists will fail.** Use `unified_panel` for raw, the daily tables for normalized.
2. **`country_reference` = 31 rows, not 34.** Counting "countries" from it is 3 short (sleeves ChinaH/NASDAQ/US SmallCap are folded in).
3. **`unified_panel` = 43 countries, not 34** (demographics_dip injects 9 non‑T2 names: Austria, Belgium, Finland, Greece, Ireland, NZ, Norway, Portugal, Russia). A `GROUP BY country` silently includes them.
4. **`unified_panel` date range 1950 → 2100 is a trap** — the extremes are demographics projections (to 2100) and IMF WEO forecasts (to 2031), not observed data. Filter `date <= CURRENT_DATE` for realized history.
5. **AGENTS.md says unified_panel "joins 7 tables"; the NL‑Query PRD cites "1.9 M rows / 332 vars"; README cites "124 events" and "17,392,079 rows."** All stale — live is **10 tables, 17.4 M rows, 2,026 vars, 146 events.** Trust the DDL / live counts.
6. **DATA_DICTIONARY.md predmkt section is stale** (says 2026‑05‑17 / 42 signal rows; live is 2026‑05‑19 / 40).
7. **`docs/gdelt_deep_ingest_plan.md` is "PLAN ONLY"** but the theme/gcam features it describes are **already** in `gdelt_panel` (folded in, not in a separate `gdelt_deep_factors` table).

### Frequency / naming traps
8. **`gdelt_panel` is MONTHLY** despite sitting next to two daily tables.
9. **Daily vs monthly factor naming differs** (`20DTR`/`120DTR`/`120‑5DTR` vs `1MTR`/`12MTR`/`12‑1MTR`). Use `variable_meta.monthly_equivalent`.
10. **GDELT `*_rank_pct` vars are NOT 0–1 percentiles** — they were z‑scored by ASADO normalization (mean≈0, std≈0.985).
11. **GDELT long theme/gcam names are clipped to ~31 chars** before `_CS/_TS`, with trailing‑digit disambiguation (`..._sha_CS` vs `..._sha1_CS` are different themes).
12. **WB commodity `yoy_pct` and `ret_12m_pct` are numerically identical** (an alias pair broadcast as both `_YOY` and `_RET_12M`).
13. **`iNATGAS` is an index mislabeled as a price** (lives in `wb_commodity_prices`, `series_type='price'`).

### Value‑scale / data‑quality surprises
14. **Mixed units in one column:** `wb_commodity_prices.nominal_price_usd` spans $/bbl, $/mt, $/kg, $/troy oz…; factor_panel `level` mixes prices and 2010=100 indices (0.03–52,179). **Always filter by `feature`/commodity before aggregating.**
15. **Broadcast vars are global** — a naive `GROUP BY country` on WB commodities or FRED markets 34×‑counts the same number.
16. **`BBG_M2_YoY` for Vietnam hits 110,003** (units/parse error, 2018‑12→2019‑04).
17. **`UNDP_HDI` (should be 0–1) has rows of 93–130** for several EM countries at 2023‑12 (parse error; column avg inflated to 2.075).
18. **`MS_CentralBank_*_GDP` fields are not clean %‑of‑GDP** (values to 363,607; USD/scaling artifacts).
19. **`MS_ETF_NetFlow_to_MarketCap` / `MS_Passive_AUM_to_MarketCap` are raw fractions (~1e‑7)** — display as 0.0 rounded, not empty.
20. **`z_36m` is winsorized at ±5.916** — extreme commodity shocks are capped.
21. **GDELT tone clusters near 0** (avg −0.78), not the nominal ±100; `country_news_sentiment` is the z‑scored version, `_raw` is the raw tone. `local_tone` only 63.8% non‑null (smaller countries).

### Coverage / completeness
22. **`factor_top20_membership` encodes only the long side** (weight>0); weak for short selection (PRD applies a 0.50 confidence penalty when used as a short fallback).
23. **`country_factor_attribution` joins MONTHLY only** — no daily attribution; top‑20% bucket only, not full portfolio, not causal.
24. **`event_log` has no 'low' severity and no `end_date`** — multi‑day overlap matching degenerates to point dates. `events_in_window(country=…)` returns country‑specific **OR** global events unless `strict_country=True`.
25. **Bloomberg coverage is very uneven** (`BBG_Debt_GDP_Ratio` = 1 country). Source tag is always `bloomberg` — can't distinguish raw `BBG_*` from derived `MS_*` except by name prefix.
26. **Stale/snapshot series hiding in live tables:** `EIA_Petroleum_Consumption_TBPD` (→2019), `UNDP_*`/`NDGAIN_*` (→2023), `OFAC_*` & 3 ETF‑fee fields (single date 2026‑05‑01), `ECB_FX_TWD_EUR` (→2020‑10). `t2_levels_daily` lags the signals table by ~2 weeks.
27. **`demographics_dip` uses a different 34‑country universe** than T2 — breaks the "all panels share the T2 names" assumption.
28. **`bilateral_portfolio_matrix`**: ISO3 keys (join via `country_reference`); `is_official_sector` always FALSE; not in `unified_panel`.

### Prediction markets (whole subsystem)
29. **Thin pilot, not production:** 1 snapshot (2026‑05‑19), 15 markets, empty resolutions. `confidence_score=0.6` is a placeholder, not accuracy.
30. **3 orphan spillover markets** (Russia‑Ukraine ceasefires, Iran peace deal) exist in `predmkt_country_spillover` + YAML but not in `predmkt_daily`/`_market_meta` — joins silently drop them; this is why `regional_conflict_premium_eastern_europe` is empty.
31. **Spillover covers only 18 of 34 countries.** `tariff_intensity_by_country` is intensity, not P(agreement). Elasticity sign = "YES lifts equity" (risk composite is sign‑flipped so positive=downside).
