# ASADO Database Map

**File:** `Data/asado.duckdb` (DuckDB v1.5, 2.7 GB) В· **35 objects** (32 base tables + 3 views) В· **~79.0 M base-table rows** В· **31вЂ“43 countries** depending on table В· **history 1950 в†’ 2100** (most realized data 2000 в†’ 2026вЂ‘05).

**ASADO** is a hybrid dataвЂ‘collection and research platform for a fixed **34вЂ‘country macro/equity "T2 universe."** It pulls macro, governance, risk, climate, trade, commodity, newsвЂ‘sentiment, predictionвЂ‘market and BloombergвЂ‘terminal data from ~26 free sources + the World Bank "Pink Sheet" + 28 Bloomberg variables, aligns everything to the 34 T2 country names, and stores it in a hybrid **DuckDB (analytical warehouse) + Neo4j (knowledge graph)** stack. The purpose is **country rotation / factor timing / regime analysis**: explanatory features feed optimizer pipelines that produce topвЂ‘20%вЂ‘ofвЂ‘countries factorвЂ‘portfolio returns. The governing doctrine is **"returns are the source of truth"** вЂ” commodity, GDELT, Bloomberg, macro, predictionвЂ‘market and graph layers are *context* unless explicitly joined back to a return surface.

> вљ пёЏ **Generated by automated deepвЂ‘dive (2026вЂ‘05вЂ‘29).** Every count, date range and example below was verified against the live `asado.duckdb` at that time. Where the project docs (README/CLAUDE.md/PRDs) disagree with the live DB, this map trusts the **live DB** and flags the mismatch. See [В§5 Gotchas](#5-gotchas--caveats).

## 1. Quick orientation

### The tidy/long panel pattern
Nearly every table shares one schema:

| Column | Type | Example |
|---|---|---|
| `date` | DATE | `2020-01-01` (firstвЂ‘ofвЂ‘month for monthly data) |
| `country` | VARCHAR | `Brazil` (exact T2 name) |
| `value` | DOUBLE | `189.37` |
| `variable` | VARCHAR | `Best PE _CS`, `BIS_Credit_GDP_Gap`, `WB_CMDTY_CRUDE_BRENT_LEVEL` |
| `source` | VARCHAR | `t2`, `gdelt`, `bloomberg`, `wb_commodity`, вЂ¦ |

This is an EAV/panel layout: instead of 2,000 columns there are **2,000+ distinct `variable` values** in long form. To "see what a table holds," `SELECT DISTINCT variable` вЂ” not the column list.

### `_CS` vs `_TS` вЂ” the central concept
Every normalized signal exists in two flavors:
- **`_CS` (CrossвЂ‘Sectional):** on each **date**, zвЂ‘score the variable **across the 34 countries**. "How attractive is this country vs its peers right now." Scale: mean в‰€ 0, std в‰€ 0.98вЂ“1.0.
- **`_TS` (TimeвЂ‘Series):** for each **country**, zвЂ‘score against an **expanding window of its own history** (min 12 obs). "How extreme is this reading vs this country's own norms." Scale slightly offвЂ‘center (expanding stats drift).
- **Sign convention:** ~19 "lowerвЂ‘isвЂ‘better" variables (valuation ratios, `REER`, `RSI14`, `Debt to GDP`, shortвЂ‘term reversal, `Bloom Country Risk`, вЂ¦) are multiplied by в€’1 *after* normalization, so **higher normalized value = more attractive, universally.** A `0` often means "no signal" (zeroвЂ‘std or postвЂ‘fill imputation), **not** "neutral midвЂ‘rank."

### The 34 T2 names (exact strings required everywhere)
```
Australia, Brazil, Canada, Chile, ChinaA, ChinaH, Denmark, France, Germany,
Hong Kong, India, Indonesia, Italy, Japan, Korea, Malaysia, Mexico, NASDAQ,
Netherlands, Philippines, Poland, Saudi Arabia, Singapore, South Africa,
Spain, Sweden, Switzerland, Taiwan, Thailand, Turkey, U.K., U.S.,
US SmallCap, Vietnam
```
Mapping rules: "China" в†’ **both ChinaA and ChinaH**; "United States" в†’ **U.S., NASDAQ, AND US SmallCap**. `country_reference` collapses these to **31 sovereigns** (sleeves folded in); `daily_calendar` and the factor tables carry all **34**.

### The signal в†’ return в†’ attribution flow
```
raw levels (t2_raw)  в”Ђв”Ђnormalize (CS+TS)в”Ђв”Ђв–¶  signals (t2_master / t2_factors_daily)
                                                   в”‚  optimizer ranks countries, topвЂ‘20% bucket
                                                   в–ј
   factor_returns (monthly P&L) в—Ђв”Ђв”Ђв”¬в”Ђв”Ђв–¶ factor_top20_membership (which countries are long)
   factor_returns_daily (daily P&L)в”‚
                                   в–ј  JOIN on (date, factor, source)
                          country_factor_attribution (VIEW): contribution = weight Г— factor_return
```

### A. T2 Factor Core вЂ” the spine

The T2 system is a multiвЂ‘factor countryвЂ‘equity allocation model. Flow: **raw Bloomberg levels в†’ normalized signals в†’ factorвЂ‘portfolio returns в†’ country attribution.** Universe: exactly **34 fixed countries**, 2000вЂ‘present, **no entry/exit** (survivorshipвЂ‘bias caveat; early EM history forwardвЂ‘filled).

| Object | Type | Rows | Vars | Date range | Grain (1 row =) |
|---|---|---:|---:|---|---|
| `t2_raw` | TABLE | 485,582 | 53 | 2000вЂ‘02 в†’ 2026вЂ‘05 | dateГ—countryГ—variable, **raw monthly level** |
| `t2_master` | TABLE | 1,192,584 | 111 | 2000вЂ‘02 в†’ 2026вЂ‘05 | dateГ—countryГ—variable, **normalized + forward returns** |
| `t2_factors_daily` | TABLE | 32,340,392 | 109 | 2000вЂ‘01 в†’ 2026вЂ‘05вЂ‘07 | dateГ—countryГ—variable, **daily signal** |
| `t2_levels_daily` | TABLE | 13,698,294 | 47 | 2000вЂ‘01 в†’ 2026вЂ‘04вЂ‘21 | dateГ—countryГ—variable, **raw daily level** |
| `factor_returns` | TABLE | 277,116 | 1,678 (3 sources) | 2000вЂ‘02 в†’ 2026вЂ‘04 | dateГ—factorГ—source, **monthly portfolio return** |
| `factor_returns_daily` | TABLE | 1,293,492 | 178 (2 sources) | 2000вЂ‘01 в†’ 2026вЂ‘05вЂ‘07 | dateГ—factorГ—source, **daily portfolio return** |
| `factor_top20_membership` | TABLE | 2,104,980 | 1,681 | 2000вЂ‘02 в†’ 2026вЂ‘05 | dateГ—countryГ—factorГ—source, **weight>0 only** |
| `country_factor_attribution` | VIEW | 2,087,921 | вЂ” | 2000вЂ‘02 в†’ 2026вЂ‘04 | dateГ—countryГ—factorГ—source, **contribution** |
| `t2_factors_monthly_from_daily` | VIEW | 1,039,973 | 109 | 2000вЂ‘01 в†’ 2026вЂ‘05 | monthГ—countryГ—variable EOM snapshot |

- **`t2_raw`** вЂ” raw Bloomberg/workbook monthly levels (53 vars), e.g. `Best PE` (0.05вЂ“137), `Inflation` (в€’5.3 to 37), `REER` (64вЂ“186), `RSI14` (18вЂ“95), `1MTR` (trailing 1вЂ‘mo return, в€’0.41 to +0.49). Raw inputs, *not* normalized.
- **`t2_master`** вЂ” the processed canonical panel: 111 vars = **5 forward returns** (`1MRet, 3MRet, 6MRet, 9MRet, 12MRet`) + **53 `_CS`** + **53 `_TS`**. Raw levels are dropped (only `Best PE _CS/_TS`, no bare `Best PE`). **вљ пёЏ ForwardвЂ‘return trap:** `1MRet` at month *t* = raw `1MTR` at month *t+1* (verified). It is the *forward* dependent variable, not a sameвЂ‘period return.
- **`t2_factors_daily`** вЂ” daily normalized signals (the eventвЂ‘study workhorse): 52 `_CS` + 52 `_TS` + 5 daily returns (`1DRet, 5DRet, 20DRet, 60DRet, 120DRet`). Daily naming differs from monthly (`20DTR` vs `1MTR`, `120DTR` vs `12MTR`, `120вЂ‘5DTR` vs `12вЂ‘1MTR`); use `variable_meta.monthly_equivalent` to map.
- **`t2_levels_daily`** вЂ” daily **raw** levels (47 vars) when you need actual prices/ratios not zвЂ‘scores: `PX_LAST`, `RSI14`, `REER`, `10Yr Bond` (yields %), `Inflation`, etc. ~2 weeks staler than the signals table (max 2026вЂ‘04вЂ‘21).
- **`factor_returns`** (monthly realized P&L, schema `date,factor,value,source`) вЂ” longвЂ‘only topвЂ‘20%вЂ‘bucket portfolio return per factor. **3 sources:** `t2_optimizer` (82 factors, T2 style), `econ_optimizer` (274 factors, BIS/macro names like `BIS_Credit_GDP_Gap_CS`), `gdelt_optimizer` (1,322 factors, news names). Value в‰€ percentвЂ‘scale (std ~2, range В±15).
- **`factor_returns_daily`** вЂ” daily P&L, **2 sources only:** `t2_optimizer_daily` (104 factors) + `gdelt_optimizer_daily` (74). **No econ daily.** Value в‰€ dailyвЂ‘scale (std ~1 / ~0.36).
- **`factor_top20_membership`** вЂ” sparse (weight>0 only): which countries are **long** each factor and their weight (sum в‰€ 1.0 per dateГ—factor; soft 15вЂ“25% rank taper from "Step Three"). ~7 countries long per T2 factor. **Encodes only the long side вЂ” not shorts.**
- **`country_factor_attribution`** (VIEW) вЂ” `contribution = weight Г— factor_return`, joining membership to **monthly** `factor_returns` (verified: 0 of 2.09 M rows deviate). TopвЂ‘20% bucket only, **not** a full portfolio decomposition and **not** causal. No daily attribution surface exists.

**Removed concept (2026-06-10):** the "8 live optimizer-selected factors" list (from `T2_Optimizer_Top.xlsx`) was a stale artifact of the retired Fuzzy Daily project. The `is_optimizer_selected` flag has been dropped from `variable_meta` and Neo4j; nothing in the warehouse designates a live factor whitelist.

### B. GDELT News / Sentiment

CountryвЂ‘day news sentiment & attention from GDELT's Global Knowledge Graph (GKG v2), built by an **external** pipeline and ingested via `setup_duckdb.py` (no ASADO collector runs GDELT). Three tables:

| Table | Rows | Freq | Vars | Countries | Date range |
|---|---:|---|---:|---|---|
| `gdelt_raw_daily` | 943,652 | daily | 45 **wide cols** | **249** (raw GDELT entities) | 2015вЂ‘02вЂ‘18 в†’ 2026вЂ‘05вЂ‘07 |
| `gdelt_factors_daily` | 10,085,794 | daily | 75 (tidy) | 34 (T2) | 2015вЂ‘06вЂ‘24 в†’ 2026вЂ‘05вЂ‘08 |
| `gdelt_panel` | 5,622,818 | **MONTHLY** | **1,323** | 34 (T2) | 2015вЂ‘09вЂ‘01 в†’ 2026вЂ‘03вЂ‘01 |

- **`gdelt_raw_daily`** (WIDE, not tidy) вЂ” one row per countryвЂ‘day, 45 columns: volume/attention (`n_articles`, `local/foreign_n_articles`, `local_attention_share`), tone (`tone_mean` avg в€’0.78, `tone_p10/50/90`, `tone_dispersion`), localвЂ‘vsвЂ‘foreign tone split, themes (`top_themes_json`, often empty), **GKG fetchвЂ‘completeness QA columns** (`gkg_fetch_share` avg 0.998, `day_status` ok/partial/NULL), and preвЂ‘computed signals (`country_news_sentiment`, `country_news_risk`, `attention_shock`, zвЂ‘scores). This is the **249вЂ‘country offвЂ‘universe bridge** (incl. Iran, North Korea, etc.); keyed by FIPS `country_code_gdelt` + `country_iso3`, China = single `CHN` row (no sleeve fanout yet).
- **`gdelt_factors_daily`** вЂ” 75 tidy daily signals = 37 base names Г— `_CS`/`_TS` + `1DRet`. Families: sentiment (fast/slow/trend + z), attention (fast/slow/trend, shock), risk/dispersion, localвЂ‘foreign tone split. **Headline composites:** `monthly_metronome` = 0.35В·sent_fast_z + 0.20В·sent_slow_z + 0.20В·sent_trend_z + 0.15В·attn_fast_z в€’ 0.10В·risk_fast_z (higher = more positive outlook); `monthly_risk` = the riskвЂ‘off mirror; `monthly_defensive` = в€’`monthly_risk`. EWMA spans: fast=5, slow=20, risk=10.
- **`gdelt_panel`** вЂ” **MONTHLY** (firstвЂ‘ofвЂ‘month), the *only* table with the deep feature families: `theme_*` (1,032 GKG themeвЂ‘share vars, e.g. `theme_PROTEST_share`, `theme_ARMEDCONFLICT_share`) and `gcam_*` (150 GDELT emotional/psychometric dictionary vars: VADER, LoughranвЂ‘McDonald, Moral Foundations, ANEW, etc.). Plus the `core_signal` family (140) and `1MRet`. Structure: 661 `_CS` + 661 `_TS` + `1MRet`.

**`1DRet`/`1MRet` are bitвЂ‘exact aliases of the T2 country returns** (copied in as the optimizer's dependent variable) вЂ” **not** an independent return source. The actual GDELTвЂ‘optimizer P&L lives in `factor_returns_daily` (source `gdelt_optimizer_daily`).

### C. World Bank Commodities

Built from the official World Bank "Pink Sheet" workbook (`CMO-Historical-Data-Monthly.xlsx`, source dated 2026вЂ‘05вЂ‘04). Chain: **raw prices/indices в†’ meta (catalog + projection flag) в†’ features (7 trailing transforms) в†’ factor_panel (broadcast to 34 countries).** All span **1960вЂ‘01 в†’ 2026вЂ‘04** (monthly). **ExplanatoryвЂ‘only by design** вЂ” never fed into optimizerвЂ‘output tables.

| Table | Rows | Grain | Notes |
|---|---:|---|---|
| `wb_commodity_prices` | 50,099 | dateГ—commodity_code | 71 benchmark **nominalвЂ‘USD** prices, 10 categories, 0 NULLs |
| `wb_commodity_indices` | 12,736 | dateГ—index_code | 16 aggregate indices (**2010=100**), e.g. iENERGY, iAGRICULTURE |
| `wb_commodity_meta` | 87 | series_code | catalog; `is_projected_to_factor_panel` = the broadcast switch |
| `wb_commodity_features` | 435,618 | dateГ—series_codeГ—feature | 7 trailing transforms, commodity axis preserved |
| `wb_commodity_factor_panel` | 9,600,274 | dateГ—countryГ—variable | 371 vars broadcast to 34 countries |

- **7 features:** `level`, `mom_pct`, `yoy_pct`, `ret_3m_pct`, `ret_12m_pct`, `vol_12m`, `z_36m` (z winsorized at В±5.916). PointвЂ‘inвЂ‘time safe (trailing only).
- **Broadcast math:** 53 projected series (all 16 indices + 37 highвЂ‘signal prices) Г— 7 features = **371 variables**, each **identical across all 34 countries** for a given date (`is_global_broadcast=TRUE` for 100% of rows). These are global prints, not countryвЂ‘specific numbers.
- **Naming:** `WB_CMDTY_{CODE}_{SUFFIX}` with irregular suffixes (`mom_pctв†’_MOM`, `yoy_pctв†’_YOY`, `ret_12m_pctв†’_RET_12M`, вЂ¦). E.g. `WB_CMDTY_CRUDE_BRENT_LEVEL`, `WB_CMDTY_COPPER_YOY`.
- **Footprint:** these 371 vars are **55.1% of all unified_panel rows** and the largest block of `variable_meta` (371/654 = 56.7%).

### F. Reference / Meta / Events / Calendar / Unified Surface

The connective tissue.

- **`country_reference`** (31 rows) вЂ” ISO/sourceвЂ‘code crosswalk. **31 sovereigns, not 34:** ChinaH folds into ChinaA, NASDAQ + US SmallCap fold into U.S. (only those 2 parent rows have `has_market_sleeves=TRUE`; sleeves live in the `market_sleeve_variants`/`all_country_variants` columns). ChinaA is the one `graph_role='sovereign_proxy'`. Columns: `iso2, iso3, imf, oecd, bis, wb, fred_epu, gpr, epu_col, вЂ¦`. Static dimension (no date). Join on `country` (T2 name) or `iso3`.
- **`variable_meta`** (654 rows) вЂ” the data dictionary for the **daily + commodity + predmkt** surfaces only (wb_commodity 371, t2_factors_daily 109, gdelt_factors_daily 75, t2_levels_daily 47, gdelt_raw_daily 38, predmkt_signals_daily 14). Columns: `variable, source_table, source_file, native_frequency (only 'monthly'|'D'), monthly_equivalent, is_normalized, category, freshness_expectation`. (`is_optimizer_selected` was dropped 2026-06-10 вЂ” stale Fuzzy Daily artifact.) `monthly_equivalent` is meaningful in only 8 rows (dailyв†’monthly return map); for commodities it just echoes the variable name. **Does NOT cover** the 7 monthly macro tables or `gdelt_panel`.
- **`daily_calendar`** (327,216 rows) вЂ” perвЂ‘country tradingвЂ‘day flags, 34 countries, 2000вЂ‘01 в†’ 2026вЂ‘05. Stores **every** calendar day (`is_trading_day` TRUE 222,076 / FALSE 105,140). Anchors event windows & monthвЂ‘end snapshots.

## 4. Glossary of the critical conventions

- **`_CS`** = crossвЂ‘sectional zвЂ‘score (across countries, per date). **`_TS`** = timeвЂ‘series zвЂ‘score (within country, expanding history).
- **Forward return** (`1MRet`вЂ¦`12MRet` in t2_master): `1MRet@t = 1MTR@t+1`. The dependent variable, not trailing.
- **topвЂ‘20% bucket**: each factor's long portfolio = the mostвЂ‘attractive ~15вЂ“25% of countries (soft taper), weights sum to 1.
- **broadcast / global variable**: same value for all countries on a date (all WB commodities, FRED markets, `Global_GPR*`). Never treat as countryвЂ‘specific.
- **returnsвЂ‘first guardrail**: optimizer *outputs* (factor_returns*) are never unioned into the input surfaces вЂ” prevents an input/output modeling cycle.

## 5. Gotchas & caveats (consolidated)

### DocвЂ‘vsвЂ‘reality mismatches (the docs are partly stale)
1. **`feature_panel` and `normalized_panel` DO NOT EXIST in the live DB** вЂ” yet CLAUDE.md/README call `feature_panel` (claimed 31.6 M rows / 3,048 vars) "the most commonly queried" surface. The only `%panel%` objects are `unified_panel` (VIEW), `gdelt_panel`, `wb_commodity_factor_panel`. **Any query/code assuming `feature_panel` exists will fail.** Use `unified_panel` for raw, the daily tables for normalized.
2. **`country_reference` = 31 rows, not 34.** Counting "countries" from it is 3 short (sleeves ChinaH/NASDAQ/US SmallCap are folded in).
3. **`unified_panel` = 43 countries, not 34** (demographics_dip injects 9 nonвЂ‘T2 names: Austria, Belgium, Finland, Greece, Ireland, NZ, Norway, Portugal, Russia). A `GROUP BY country` silently includes them.
4. **`unified_panel` date range 1950 в†’ 2100 is a trap** вЂ” the extremes are demographics projections (to 2100) and IMF WEO forecasts (to 2031), not observed data. Filter `date <= CURRENT_DATE` for realized history.
5. **AGENTS.md says unified_panel "joins 7 tables"; the NLвЂ‘Query PRD cites "1.9 M rows / 332 vars"; README cites "124 events" and "17,392,079 rows."** All stale вЂ” live is **10 tables, 17.4 M rows, 2,026 vars, 146 events.** Trust the DDL / live counts.
6. **DATA_DICTIONARY.md predmkt section is stale** (says 2026вЂ‘05вЂ‘17 / 42 signal rows; live is 2026вЂ‘05вЂ‘19 / 40).
7. **`docs/gdelt_deep_ingest_plan.md` is "PLAN ONLY"** but the theme/gcam features it describes are **already** in `gdelt_panel` (folded in, not in a separate `gdelt_deep_factors` table).

### Frequency / naming traps
8. **`gdelt_panel` is MONTHLY** despite sitting next to two daily tables.
9. **Daily vs monthly factor naming differs** (`20DTR`/`120DTR`/`120вЂ‘5DTR` vs `1MTR`/`12MTR`/`12вЂ‘1MTR`). Use `variable_meta.monthly_equivalent`.
10. **GDELT `*_rank_pct` vars are NOT 0вЂ“1 percentiles** вЂ” they were zвЂ‘scored by ASADO normalization (meanв‰€0, stdв‰€0.985).
11. **GDELT long theme/gcam names are clipped to ~31 chars** before `_CS/_TS`, with trailingвЂ‘digit disambiguation (`..._sha_CS` vs `..._sha1_CS` are different themes).
12. **WB commodity `yoy_pct` and `ret_12m_pct` are numerically identical** (an alias pair broadcast as both `_YOY` and `_RET_12M`).
13. **`iNATGAS` is an index mislabeled as a price** (lives in `wb_commodity_prices`, `series_type='price'`).

### ValueвЂ‘scale / dataвЂ‘quality surprises
14. **Mixed units in one column:** `wb_commodity_prices.nominal_price_usd` spans $/bbl, $/mt, $/kg, $/troy ozвЂ¦; factor_panel `level` mixes prices and 2010=100 indices (0.03вЂ“52,179). **Always filter by `feature`/commodity before aggregating.**
15. **Broadcast vars are global** вЂ” a naive `GROUP BY country` on WB commodities or FRED markets 34Г—вЂ‘counts the same number.
16. **`BBG_M2_YoY` for Vietnam hits 110,003** (units/parse error, 2018вЂ‘12в†’2019вЂ‘04).
17. **`UNDP_HDI` (should be 0вЂ“1) has rows of 93вЂ“130** for several EM countries at 2023вЂ‘12 (parse error; column avg inflated to 2.075).
18. **`MS_CentralBank_*_GDP` fields are not clean %вЂ‘ofвЂ‘GDP** (values to 363,607; USD/scaling artifacts).
19. **`MS_ETF_NetFlow_to_MarketCap` / `MS_Passive_AUM_to_MarketCap` are raw fractions (~1eвЂ‘7)** вЂ” display as 0.0 rounded, not empty.
20. **`z_36m` is winsorized at В±5.916** вЂ” extreme commodity shocks are capped.
21. **GDELT tone clusters near 0** (avg в€’0.78), not the nominal В±100; `country_news_sentiment` is the zвЂ‘scored version, `_raw` is the raw tone. `local_tone` only 63.8% nonвЂ‘null (smaller countries).

### Coverage / completeness
22. **`factor_top20_membership` encodes only the long side** (weight>0); weak for short selection (PRD applies a 0.50 confidence penalty when used as a short fallback).
23. **`country_factor_attribution` joins MONTHLY only** вЂ” no daily attribution; topвЂ‘20% bucket only, not full portfolio, not causal.
24. **`event_log` has no 'low' severity and no `end_date`** вЂ” multiвЂ‘day overlap matching degenerates to point dates. `events_in_window(country=вЂ¦)` returns countryвЂ‘specific **OR** global events unless `strict_country=True`.
25. **Bloomberg coverage is very uneven** (`BBG_Debt_GDP_Ratio` = 1 country). Source tag is always `bloomberg` вЂ” can't distinguish raw `BBG_*` from derived `MS_*` except by name prefix.
26. **Stale/snapshot series hiding in live tables:** `EIA_Petroleum_Consumption_TBPD` (в†’2019), `UNDP_*`/`NDGAIN_*` (в†’2023), `OFAC_*` & 3 ETFвЂ‘fee fields (single date 2026вЂ‘05вЂ‘01), `ECB_FX_TWD_EUR` (в†’2020вЂ‘10). `t2_levels_daily` lags the signals table by ~2 weeks.
27. **`demographics_dip` uses a different 34вЂ‘country universe** than T2 вЂ” breaks the "all panels share the T2 names" assumption.
28. **`bilateral_portfolio_matrix`**: ISO3 keys (join via `country_reference`); `is_official_sector` always FALSE; not in `unified_panel`.

### Prediction markets (whole subsystem)
29. **Thin pilot, not production:** 1 snapshot (2026вЂ‘05вЂ‘19), 15 markets, empty resolutions. `confidence_score=0.6` is a placeholder, not accuracy.
30. **3 orphan spillover markets** (RussiaвЂ‘Ukraine ceasefires, Iran peace deal) exist in `predmkt_country_spillover` + YAML but not in `predmkt_daily`/`_market_meta` вЂ” joins silently drop them; this is why `regional_conflict_premium_eastern_europe` is empty.
31. **Spillover covers only 18 of 34 countries.** `tariff_intensity_by_country` is intensity, not P(agreement). Elasticity sign = "YES lifts equity" (risk composite is signвЂ‘flipped so positive=downside).
