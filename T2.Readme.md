# T2 Factor Timing — Database & Transformation Reference

**Last updated:** 2026-04-10  
**Purpose:** Describes the raw data stored in `T2 Master.xlsx`, the transformations applied by `Step Two Create Normalized Tidy.py`, and the structure of the two output files (`Normalized_T2_Master.xlsx` and `Normalized_T2_MasterCSV.csv`).

## 1. Source Database — `T2 Master.xlsx`

`T2 Master.xlsx` is the consolidated, cleaned master dataset for the T2 Factor Timing system. It is produced by **Step One** (`Step One Create T2Master.py`) from raw Bloomberg multi-sheet extracts, and is the **sole input** to the Step Two normalization pipeline.

## 2. Step Two Transformation — `Step Two Create Normalized Tidy.py`

### 2.1 Purpose

Step Two reads every sheet from `T2 Master.xlsx` and produces two normalized variants of each financial variable:

1. **Cross-Sectional (CS)** — ranks within a date (how does this country compare to others right now?)
2. **Time-Series (TS)** — ranks within a country's own history (how does this reading compare to norms for this country?)

Return sheets (`1MRet` through `12MRet`) are copied through unchanged.

### 2.2 Pipeline Flow

```
T2 Master.xlsx
      │
      ▼
 ┌─────────────────────────────────────────────────────┐
 │  For each sheet (variable):                          │
 │                                                      │
 │  IF sheet in [1MRet, 3MRet, 6MRet, 9MRet, 12MRet]: │
 │      → Copy directly (no normalization)              │
 │                                                      │
 │  ELSE:                                               │
 │      → Cross-sectional normalize  → sheet_{var}_CS  │
 │      → Time-series normalize      → sheet_{var}_TS  │
 └─────────────────────────────────────────────────────┘
      │                               │
      ▼                               ▼
Normalized_T2_Master.xlsx    Normalized_T2_MasterCSV.csv
  (wide format, 111 sheets)    (long/tidy format, 4 columns)
```

### 2.3 Preprocessing (applied to all sheets)

Before normalization, the following steps are applied to every sheet:

1. **Column rename:** The first column is renamed to `date`.
2. **Date parsing:** Dates are parsed with `pd.to_datetime(..., errors='coerce')`. Rows where the date cannot be parsed are dropped.
3. **Month normalization:** All dates are converted to the **first-of-month** timestamp using `.dt.to_period('M').dt.to_timestamp()`.
4. **Set index:** `date` becomes the DataFrame index.

### 2.4 Normalization Methods

#### Cross-Sectional Normalization (suffix `_CS`)

For each **date** (row), normalize across all **countries** (columns):

```
CS_score(country, date) = (value(country, date) − mean_t(date)) / std_t(date)
```

Where:

- `mean_t(date)` = mean across all 34 countries at that point in time
- `std_t(date)` = standard deviation across all 34 countries at that point in time (N−1 Bessel correction, pandas default)

**Interpretation:** A CS score of +1.5 means a country is 1.5 standard deviations richer on that factor relative to the cross-section of countries on that date.

#### Time-Series Normalization (suffix `_TS`)

For each **country** (column), normalize over time using an **expanding window**:

```
TS_score(country, date) = (value(country, date) − expanding_mean(country, date)) 
                           / expanding_std(country, date)
```

Where `expanding_mean` and `expanding_std` use all observations from the start of history **up to and including** the current date (i.e., no look-ahead bias).

- **Minimum periods:** 12 months of history are required before a TS score is computed; early observations will be `NaN`.
- **Zero-std handling:** If the expanding standard deviation is 0 (e.g., US currency vs USD is always 0), the normalized value is set to `0` rather than `NaN` or `inf`.

**Interpretation:** A TS score of +2.0 means a country's current reading is 2 standard deviations above its own historical average up to that point.

### 2.5 Inversion — "Lower is Better" Variables

For variables where a lower raw value signals a more attractive opportunity (e.g., lower P/E = cheaper), the normalized score is **multiplied by −1** after normalization, so that the resulting score is still positive when the variable is in a favorable regime:

```python
normalized_score = −1 × normalized_score    # for inverted variables only
```

### 2.6 Missing Data Handling


| Step                        | Treatment                                                                                  |
| --------------------------- | ------------------------------------------------------------------------------------------ |
| Forward-fill within country | Gaps in a country's series are filled with the last known value before normalization       |
| After forward-fill          | Any remaining `NaN` values (e.g., at the very start of a country's history) are set to `0` |
| Logged                      | Missing data incidents are printed to console during processing                            |

## 4. Key Design Decisions


| Decision                                  | Rationale                                                                                                                                                                |
| ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Expanding window for TS** (not rolling) | Avoids parameter selection for window length; uses all available history at each point, growing as data accumulates                                                      |
| **Cross-sectional normalization**         | Removes the level of a variable at each point in time, leaving only relative cross-country signal; compensates for mean-regime shifts over decades                       |
| **Inversion convention**                  | Ensures all normalized columns share a single semantic: **higher = better**. Downstream steps can sum or weight any combination of factors without sign confusion        |
| **Return series pass-through**            | Returns are the dependent variable (or used in portfolio construction); normalizing them would destroy the economic content (e.g., 5% return ≠ the same as a +1 z-score) |
| **Month normalization to first-of-month** | Ensures consistent merging and time-alignment across variables that may arrive on different business days within a month                                                 |
| **min_periods=12 for TS**                 | At least 1 full year of history is required before a TS z-score is meaningful; avoids spurious scores in the early ramp-up of each country's history                     |
| **Zero-std → 0**                          | Prevents `inf` / `NaN` blowups when a series has no variation (e.g., US currency always = 1.0 USD)                                                                       |
| **Both wide and long output**             | Wide (Excel) is useful for inspection and row-by-row analysis per variable; long (CSV) is optimal for pandas group-by, filtering, and ML feature engineering             |

## 7. Data Quality Notes

- **Survivorship bias:** The country universe is fixed at 34 markets; no countries enter or exit. Gaps in early history for some emerging markets (e.g., Vietnam, Saudi Arabia) are forward-filled from their first available observation.
- **Outlier handling:** Outlier winsorization and forward-filling is performed upstream in Step One. Step Two operates on the already-cleaned data from `T2 Master.xlsx`.
- **Look-ahead bias:** The TS expanding window is strictly backward-looking. CS normalization is also safe as it only uses values available at that cross-section date.
- **NaN propagation:** Any `NaN` surviving after forward-fill is zeroed out, so no `NaN` values appear in the output files. Zero-imputed observations should be interpreted as "no signal" rather than "neutral signal" and treated carefully in factor timing models.

## 8. Glossary


| Term                 | Definition                                                                                                       |
| -------------------- | ---------------------------------------------------------------------------------------------------------------- |
| **CS**               | Cross-Sectional: normalized within the cross-section of countries at a single date                               |
| **TS**               | Time-Series: normalized within a country's own history up to the current date (expanding window)                 |
| **z-score**          | (value − mean) / standard deviation                                                                              |
| **Expanding window** | A cumulative window that grows as time progresses, using all data from the start through the current observation |
| **Inversion**        | Multiplying a normalized score by −1 so that higher = better for all variables                                   |
| **Tidy format**      | Long-form tabular data where each row is a single observation (date, country, variable, value)                   |
| **Wide format**      | One variable per sheet/column, countries as columns, dates as rows                                               |
| **T2**               | The T2 Factor Timing system — a multi-factor country equity allocation model                                     |
| **BEST**             | Bloomberg BEST consensus estimates (forward-looking analyst forecasts)                                           |
