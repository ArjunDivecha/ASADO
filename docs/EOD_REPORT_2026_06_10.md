## 2. `t2_levels_daily` staleness — root cause found and fixed

**Root cause:** `build_t2_master_daily.py` winsorized level series (PX_LAST,
Currency, 10Yr Bond) against their **full-sample** median ± 5×MAD. For any
trending series this is a static ceiling: SPX froze at 5331.89 from 2025-04,
TRY froze from 2020-04 — and it was lookahead besides.

**Fix (v1.1):** trailing 252-day rolling median ± 5×rolling-MAD clip. Adapts
to trends, still kills one-off data-error spikes, no lookahead.

**Verified after rebuild:** SPX now 7334.08 (was 5331.89), TRY 46.14 and
moving daily (was frozen), Korea shows the +9% June-8 bounce — 28–29 distinct
values per series since May 1.

**Bonus find from the new D4 freshness guard:** Taiwan's 10Yr Bond series has
been genuinely dead upstream **since 2020-05-20** (not a winsorize artifact —
the raw pull is static). Philippines (last change 05-15) and Vietnam (01-26)
are also stale. These are now reported loudly as DETECTOR_DEGRADED rows every
night instead of silently producing fake z-scores.

## 7. Daily harness v2 — and graph_trade_gap demoted WATCH → WEAK

v2 gives daily runs the full gate set instead of IC-only:

- **Returns:** corrected backward-labeled trading-day returns.
- **Portfolios:** overlapping-tranche construction (Jegadeesh-Titman) —
  hold_days tranches, each re-ranks on its own offset, next-day execution
  (no same-close fills). Three new execution-timing canary tests prove the
  lag is exactly one day: contemporaneous-cheat signal earns ~nothing,
  next-day-foresight signal earns hugely, two-day-foresight earns nothing.
- **Costs:** 10/25/50 bps one-way × actual turnover + 50 bps/yr borrow.
- **Deflated Sharpe:** now period-aware (252/yr for daily).
- **NW t-stat fix:** max_lag now equals the horizon in trading days; v1 used
  h//21 → lag 0 for the 5d horizon, which overstated the t-stats (7.10 → 3.05
  on the same signal family). Treat archived v1 daily t-stats accordingly.
