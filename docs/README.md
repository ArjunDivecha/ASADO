# ASADO Documentation Index

Last updated: 2026-06-17

This file tells Claude Code / Codex / any agent which docs are canonical specs,
which are generated reports, and which are historical snapshots — so you don't
accidentally treat a one-day audit as the source of truth.

## Start here (agent cold start)

1. `../README.md` — high-level architecture, quick start, current pipeline state.
2. `../CLAUDE.md` — detailed conventions: script headers, paths, logging,
   error handling, date rules, database bridge patterns, important files.
3. `../AGENTS.md` — durable learned facts and user preferences; the place to
   record recurring gotchas so future agents don't repeat them.
4. `../llmchat.md` — append-only session log: what was built, why, and the
   hard-won lessons from each major push.

Always read `AGENTS.md` before making operational changes. Read `llmchat.md`
when you need to understand the history behind a design choice.
